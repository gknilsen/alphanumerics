Resampler Class

Resampler(UInt32, UInt32)
Creates a new Resampler object which resamples the signal by a factor p/q. Default parameters are used for the lowpass filter. The default filter is a Kaiser-windowed sinc, where the alpha parameter of the Kaiser window is 5.0. THe cutoff frequency of the filter (which is applied to the signal after upsampling by p) is 1/max(p, q) of Nyquist. The filter length is 20*cutoff+1.
Resampler(UInt32, UInt32, Double)
Creates a new Resampler object with a custom filter. The cutoff of the filter should be at about 1/max(p, q) of the Nyquist frequency. See also Resampler(UInt32, UInt32)
Resampler(UInt32, UInt32, UInt32)
Creates a Resampler object with default settings except for n, which specifies the length of the anti-aliasing filter. The actual length of the filter will be 2*n*max(p, q) + 1. The filter length provides a compromise between lower computational resources and phase delay (delay only relevant for ResampleContinuous) on one hand versus better alias suppression and less attenuation of high frequencies on the other. The default value of n is 10. See also Resampler(UInt32, UInt32)
Resampler(UInt32, UInt32, Double, UInt32, Double)
Creates a Resampler object with a Kaiser-windowed sinc filter based on the given parameters. See also Resampler(UInt32, UInt32)

Resample
Resamples the signal x, compensating for delay caused by the lowpass filter. Do not use this function for real-time processing, use only if the whole signal is known at call-time.
ResampleContinuous
Resampling function which can be called over and over with new contiguous blocks of input signal. This can be used for real time processing, or for very long signals. Unlike Resample, this function does not compensate for delay caused by the filtering. The delay caused by the filter will be n output samples if q > p, and n*p/q output samples if p > q. See Resampler(UInt32, UInt32, UInt32) for a description of n.
ResetFilter
Resets the continuous reasmpling filter state.
